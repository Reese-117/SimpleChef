# Instructions

Students are required to submit a functional prototype and supporting documentation that demonstrates the design process and application of human-computer interaction principles. The prototype may be submitted as a hosted web project and demonstrated locally. If the project is hosted online, you must include the project link in your submission.

**Note:** *All groups are required to complete an in-person demonstration during instructor or TA office hours, 2:00 PM to 5:00 PM (05/05 or 05/07).* 

The documentation should include user goal analysis (description of the target users, tasks, motivations, needs, and pain points, with a brief mention of any methods used to identify them), problems in the original UI / design motivation (discussion of major usability issues in the original interface with annotated screenshots, or motivation and UX goals if the project involves a new interface), and design rationale (explanation of major design decisions and how they improve usability, supported by relevant HCI principles). The In-person demo must be during the instructor's or TA's office hours (05/05 or 05/07). 

## Grading Criteria (20 Points)
1. Functional Prototype (10)
2. In-person Demo (5)
3. Documentation (3)
4. Submission Completeness (2)
